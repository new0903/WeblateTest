Свойство display, его типы. Создание таблиц с помощью div

Синтаксис: display: block | inline | inline-block | inline-table | list-item | none | run-in | table | table-caption | table-cell | table-column-group | table-column | table-footer-group | table-header-group | table-row | table-row-group

Приложение

Свойство display (CSS) определяет тип отображения (display type) элемента, имеющий два основных свойства, определяющих генерацию боксов — внешний тип отображения определяет расположение бокса в схеме потока (flow layout) и внутренний тип отображения определяет расположение дочерних элементов бокса (бокс - это прямоугольная область, являющаяся изображением элемента). Когда вы явным образом не задаете свойство display элементу, все равно это свойство присваивается ему по умолчанию браузером. Список возможных значений этого свойства, понимаемый разными браузерами очень короткий — block, inline, list-item и none. Все остальные допустимые значения поддерживаются браузерами выборочно.

Элемент block Элемент показывается как блочный. Применение этого значения для встроенных элементов, например тега , заставляет его вести подобно блокам — происходит перенос строк в начале и в конце содержимого.

Элемент inline Элемент отображается как встроенный. Использование блочных тегов, таких как div и p, автоматически создает перенос и показывает содержимое этих тегов с новой строки. Значение inline отменяет эту особенность, поэтому содержимое блочных элементов начинается с того места, где окончился предыдущий элемент.

Элемент inline-block Это значение генерирует блочный элемент, который обтекается другими элементами веб-страницы подобно встроенному элементу. Фактически такой элемент по своему действию похож на встраиваемые элементы (вроде тега img). При этом его внутренняя часть форматируется как блочный элемент, а сам элемент — как встроенный.

Элемент inline-table Определяет, что элемент является таблицей как при использовании тега table, но при этом таблица является встроенным элементом и происходит ее обтекание другими элементами, например, текстом.

Элемент list-item Элемент выводится как блочный и добавляется маркер списка.

Элемент none Временно удаляет элемент из документа. Занимаемое им место не резервируется и веб-страница формируется так, словно элемента и не было. Изменить значение и сделать вновь видимым элемент можно с помощью скриптов, обращаясь к свойствам через объектную модель. В этом случае происходит переформатирование данных на странице с учетом вновь добавленного элемента.

Элемент run-in Устанавливает элемент как блочный или встроенный в зависимости от контекста.

Элемент table Определяет, что элемент является блочной таблицей подобно использованию тега table.

Элемент table-caption Задает заголовок таблицы подобно применению тега caption.

Элемент table-cell Указывает, что элемент представляет собой ячейку таблицы (тег td или th).

Элемент table-column Назначает элемент колонкой таблицы, словно был добавлен тег col.

Элемент table-column-group Определяет, что элемент является группой одной или более колонок таблицы, как при использовании тега colgroup.

Элемент table-footer-group Используется для хранения одной или нескольких строк ячеек, которые отображаются в самом низу таблицы. По своему действию сходно с работой тега tfoot.

Элемент table-header-group Элемент предназначен для хранения одной или нескольких строк ячеек, которые представлены вверху таблицы. По своему действию сходно с работой тега thead.

Элемент table-row Элемент отображается как строка таблицы (тег tr).

Элемент table-row-group Создает структурный блок, состоящий из нескольких строк таблицы аналогично действию тега tbody.
